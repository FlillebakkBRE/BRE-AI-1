2. **Adobe Sign API** — ⛔ BLOKKERT (bekreftet av Adobe 2026-07-07): REST-API (integrasjonsnøkkel/OAuth-app) krever **Enterprise-abonnement**. BREs nåværende Acrobat Sign Solutions (småbedrift) har IKKE API-tilgang — derfor mangler både «Integrasjonsnøkkel» og «API-applikasjoner» i UI (kun API-informasjon + API-forespørselslogg vises). Adobe tilbød å ta det på telefon m/ dedikert team. **BESLUTNING kreves:** (A) oppgradere til Adobe Enterprise (pris ukjent — Frode ringer for tilbud), eller (B) bytte til e-sign-leverandør med API på standard-tier — vurder **Verified.eu / Signere.no / Penneo (norske, BankID)** eller GetAccept; ofte billigere + bedre for norske kundesigneringer ved lavt volum/ett konto. Anbefalt: sammenlign Adobe Enterprise-pris mot Verified/Signere før binding. **ENKLERE VEI (Frode delte dev-dok 2026-07-07):** bruk **Integrasjonsnøkkel** i stedet for OAuth-app — en permanent bearer-token for ÉN konto, perfekt for BREs interne bruk, slipper OAuth-app-flyten som var blokkert. Lages i Acrobat Sign: Konto → Acrobat Sign API → API-informasjon → «Integrasjonsnøkkel» (evt. secure.echosign.com/account/echosignApi). Scopes: agreement_read/write/send + library_read. API-referanse **v6** (secure.na1.adobesign.com/public/docs/restapi/v6). Ved oppstart: kall `baseUris` for å få riktig shard-URL (BRE = eu1 → api.eu1.adobesign.com). Når nøkkel er på plass: legg i gitignorert config, bygg klient (tilbud→send til signering→status tilbake på HubSpot-deal). Postman-workspace + Java-samples finnes i Adobe-dok. **Send-flyt (bekreftet fra Adobe REST-samples 2026-07-07):** (1) POST `/transientDocuments` (last opp tilbud-PDF) → `transientDocumentId`; (2) POST `/agreements` med transientDocumentId + signatar + `signatureType:ESIGN`; (3) GET `/agreements/{id}` eller webhook → når status `SIGNED`, oppdater HubSpot-deal (flytt steg / logg notat) + hent signert PDF (`GET /agreements/{id}/combinedDocument`) og signaturlogg (`/auditTrail`) for arkiv/deal. Auth-header: `Authorization: Bearer <integrasjonsnøkkel|access_token>`. Se [[salg-pipeline]] pkt 2.
